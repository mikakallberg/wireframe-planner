In the calendar the user should either be able to create a new event by clicking the button on the left or by clicking a timeslot in the calendar. If the time slot is clicked the user is automatically given the time clicked as time planned in the add event feature, this add event tool will be found in all three resources for the user.

To form the calender structure, bordered tables will used from Bootstrap. The information in the calender will be modelled and stored in the unique API built for this project. The user will be able to plan months in advance. The checklist and notebook should probably be connected to the calender. A notification and priority will be set for the user to be able to priorities the events. The scheduling should also be adjustable through drag and drop.
